The pitch / compute

AlphaZero used 5,000 first-gen TPUs for self-play data generation and 64 TPU-v2s to train. This project uses the same network family (20- or 40-block × 256-channel ResNet, ~24–50M parameters) on one GPU, via supervised distillation from Stockfish on 30–46M positions, evaluated against a calibrated Stockfish opponent. Total cost per training run: ~16–80 GPU-hours on a single L40S — roughly three-to-four orders of magnitude less compute than the AlphaZero training run (~10³× by device-hours, ~10⁴× by FLOPs).

How it works — two phases

Training is one supervised pass; inference is MCTS using the trained net as its prior. The network alone never picks a move — MCTS does, guided by the network.

1 Training (one-shot, no self-play)

  1. Stockfish self-plays games at search-depth 15. At every position it reports its top-8 candidate moves with centipawn scores.
  2. Those scores become a soft distribution π over the 8 candidates (softmax at T = 1 pawn). The game outcome z ∈ {−1, 0, +1} is recorded from each side-to-move POV.
  3. The student trains on (state, π, z) triples — cross-entropy on the policy head against π, MSE on the value head against z. No self-play, no MCTS at training time.

2 The network

20 residual blocks × 256 channels, ~24M parameters. Input: a 19-plane board encoding (pieces, side-to-move, castling, repetitions). Output: a 4,672-dim policy-logits vector and a scalar value in [−1, +1]. The same architecture AlphaZero used — only the way the weights are obtained differs.

3 Playing a move (network + MCTS)

The trained network is a prior over moves and a value estimator for leaf positions; the move comes from Monte Carlo Tree Search on top. One root move costs N simulations (800 routine, 4,000 search-saturated). Each simulation:

  • Selection. From the root, repeatedly pick the child maximizing Q + U (avg value below the move + an exploration bonus ∝ the prior P, ∝ 1/(visits)). Selection formula (AlphaGo Zero): U(s,a) = c_puct · P(s,a) · √Σ_b N(s,b) / (1 + N(s,a)).
  • Expansion + evaluation. At an unexpanded node, encode the position, run one network forward pass, store policy logits as child priors + the value estimate.
  • Backup. Propagate the leaf value up the path, incrementing visit counts and updating Q. Sign-flip at every ply (chess alternates side-to-move).
  • Repeat N sims, then pick the move with the highest visit count (MCTS's improved policy). More sims ⇒ sharper visit distribution: the same checkpoint scores 1,807 Elo at 800 sims and 2,084 Elo at 4,000 sims.
04

Experiments

Single-variable ablations on the distilled baseline.

REJECTED

Network capacity

Doubling depth 20→40 blocks didn't help (20-block 1,810 vs 40-block 1,810, Δ ±0). 5M positions for a 24M-param net is already over-parameterized.

CONFIRMED +277

Eval-side search

Deeper MCTS at inference, no retraining (800 sims 1,807 → 4,000 sims 2,084). The "1,800 ceiling" was an eval-time artifact.

CONFIRMED +199

Data scale

6× more positions (5M 1,810 → 30M 2,009). More data beat more parameters and a stronger teacher. The two confirmed levers (search, data) are both "more compute"; the rejected lever was "smarter network" — a clean bitter-lesson result.

06

vs Leela Chess Zero

The shared shape is "a strong prior, then self-play." Lc0 is fundamentally a self-play (AlphaZero-style) engine — its strength came from years of community self-play, not from distilling a classical engine; warm-starts are accelerants. This project's twist: build the prior by distilling a classical engine (Stockfish), then try the self-play half on top. Lc0's peak is ~3,600 CCRL (top engine globally), bought by years of community self-play; the distilled prior is the practical single-GPU ceiling here.

From-scratch MuZero

Capped ~700–900 Elo (compute-starved, sparse rewards).

COLLAPSED

Distill-init MuZero

Collapsed under MCTS — reusing the 2,301 teacher's frozen representation + prediction and training only the dynamics produced latents that drift off-manifold when unrolled, so search amplified confidently-wrong rollouts (scored down to 0/30 vs Stockfish).

08

Self-play postmortem

Negative results at fixed compute.

Self-play should add +200–500 Elo on the distilled prior. At one-GPU compute it doesn't: ungated self-play regresses (→ ~1,730, a ~370-Elo drop) and gated self-play holds but doesn't climb (~2,101, gating floors regression but candidates never beat the teacher ≥55%). Six chess attempts were spot-evicted before iteration 1 finished. The binding constraint is the strength of the self-play signal at this data scale (~1.3k samples/iter), not infrastructure.

Together with MuZero, this is the project's strongest evidence: distillation ≫ self-play / from-scratch at fixed compute.

Lessons

  1. Test-before-spend — a 2-minute unit test caught an inverted MCTS value-sign that nearly cemented a false "MuZero can't learn this" conclusion.
  2. On-demand for long runs, spot only for short/restartable.
  3. Tear down idle infra (~$250 of idle EKS).
  4. Tag every instance.
  5. Reuse generated data.
